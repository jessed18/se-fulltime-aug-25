    static vs. dynamic websites

  • static websites show fixed content that only changes if the code is updated.
  • dynamic websites respond to user input or data and generate content on the fly.

    • frontend tasks

  • build the ui with html, css, and javascript.
  • make the design responsive and accessible.
  • add interactivity (buttons, forms, events).

    • backend tasks

  • backend tasks: set up and manage databases for data flow.
  • backend tasks: write server-side logic and apis.
  • backend tasks: handle security (authentication, authorization, encryption) and performance.

    • requirement analysis

  • discover and describe the problem by collecting functional (what it should do) and non-functional requirements (quality, performance, data protection). formalize them so they can be verified later.

    • languages

  • html = structure and content of the page.
  • css = style, layout, and visual quality.
  • javascript = interactivity and dynamic behavior.
